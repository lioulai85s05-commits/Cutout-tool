## 1. Brand Tone

Direction name:

- `Warm Precision`

The app should feel:

- precise
- premium
- calm
- modern
- globally understandable

The app should not feel:

- playful sticker tool
- generic AI toy
- cold blue enterprise utility
- neon cyber product

Core emotional message:

- identify clearly
- choose confidently
- extract cleanly

## 2. Product Personality

This is not a portrait-only app anymore.

It is a general object cutout tool that:

- scans a photo
- reveals selectable objects
- lets the user choose a target
- produces a clean cutout for export

The tone should communicate:

- "professional enough for work"
- "simple enough for anyone"

## 3. Visual Principle

The icon and UI must follow the same visual logic:

- a soft warm shell
- a precise selection ring
- a dark subject core
- a coral action accent

This means the brand language is:

- warm outside
- precise in the middle
- strong at the point of action

## 4. Locked Color System

Canonical colors:

- Shell Background: `#FFF5EB`
- Shell Surface: `#FFF9F1`
- Shell Surface Strong: `#FFF0E2`
- Ink Primary: `#171412`
- Ink Secondary: `#6D625A`
- Border Soft: `#E3D0C0`
- Canvas Dark: `#0B0C0E`
- Canvas Elevated: `#1D1E22`
- Accent Primary: `#FF8F69`
- Accent Deep: `#E6573D`
- Accent Soft: `#FFD8C8`

Usage rules:

- warm backgrounds for navigation and controls
- dark canvas only where image focus matters
- coral only for primary actions, active selection, and progress emphasis
- soft border tones for cards and segmented controls

## 5. Typography Tone

Typography should feel:

- clear
- quiet
- expensive

Rules:

- avoid decorative display fonts
- use clean system typography with strong hierarchy
- titles should be compact and confident
- body copy should be short and instructional

Writing tone:

- direct
- light
- not chatty
- not overexplained

## 6. Interaction Tone

The interaction should feel:

- guided
- low-friction
- reassuring

Primary flow:

1. Upload
2. Recognize
3. Select
4. Extract
5. Save

Behavior rules:

- one primary action per screen
- progress must always be visible
- active selection must be obvious
- export should feel clean and final

## 7. Recognition UI Rule

The recognition screen is the product signature.

It should use:

- dark preview stage
- multiple object outlines
- selected object highlighted in coral
- non-selected objects shown in softer contrast

Do not use:

- noisy rainbow overlays
- thick cartoon strokes
- overly technical bounding-box aesthetics

## 8. Editor UI Rule

The editor should separate image work from controls:

- image on dark canvas
- controls on warm cards
- export action in coral

The selected cutout should feel:

- isolated
- premium
- print-ready

## 9. Motion Rule

Motion should be minimal but meaningful:

- soft fade on upload
- scan sweep or ring pulse during recognition
- subtle selection confirmation when object is tapped
- no playful bounce animations

## 10. Monetization Tone

The app uses buy-once unlock logic.

This means the paywall tone should be:

- premium utility
- honest
- permanent unlock

Do not position it like:

- a subscription AI product
- a hype-driven magic tool

## 11. Execution Rule

From this point forward:

- icon work follows `Object Ring`
- UI work follows `Warm Precision`
- new screens refine this system instead of inventing new visual branches
- any future visual experiment must still map back to this color and tone system
